# PRD — 讀書筆記本系統

## 1. 專案概述
- **背景與動機**：喜愛閱讀的人往往會想要紀錄閱讀過程中的感悟或重點，但如果沒有適合的工具集中管理，這些紀錄往往變得零散。透過這款專用的讀書筆記本系統，使用者可以快速備份自己的心得，方便未來回溯。
- **目標用戶**：喜愛閱讀的讀者、學生、需要整理學習筆記以建立知識庫的使用者。
- **核心價值主張**：提供簡單、直覺的紀錄體驗，將書籍資訊（書名、評分）、個人心得有效歸檔，並支援快速的搜尋查找功能。

## 2. 功能需求
系統的五項主要功能與使用者故事：

1. **新增書籍紀錄**
   - 作為【讀者】，我希望【可以透過簡單的表單新增一本書的書名、心得以及給予評分（如1~5星）】，以便【為剛讀完的書留下紀錄】。
2. **首頁列表總覽**
   - 作為【讀者】，我希望【可以在系統的首頁看到所有我紀錄的全部書單及概略資訊（書名與評分）】，以便【快速回顧我擁有的書籍摘要】。
3. **搜尋書籍**
   - 作為【讀者】，我希望【能利用搜尋列輸入關鍵字（根據書名或心得）】，以便【從大量清單中快速找出特定的閱讀筆記】。
4. **編輯與更新筆記**
   - 作為【讀者】，我希望【能夠隨時點選進入既有的書籍紀錄進行編輯或修改評分】，以便【當我有新的感觸或想法時，能夠即時進行補充】。
5. **刪除書籍紀錄**
   - 作為【讀者】，我希望【可以刪除那些不想要或是紀錄錯誤的書籍資訊】，以便【維持版面整潔乾淨】。

## 3. 非功能需求
- **技術限制**：
  - 需使用 Flask 作為後端框架。
  - 使用 Jinja2 作為 HTML 模板引擎。
  - 使用 SQLite 儲存書籍資料。
  - 介面排版應考慮現代且簡潔的網頁設計（CSS/框架）。
- **效能與安全考量**：
  - 防止 SQL 注入攻擊（可以透過 SQLAlchemy 或參數化查詢來實踐）。
  - 對使用者輸入的心得內容防範 XSS（透過 Jinja2 預設的自動跳脫功能達成）。
  - 搜尋效能需能流暢應付千筆以內的個人書籍量。

## 4. MVP 範圍
- **Must Have (絕對需要)**：
  - 新增書籍（含書名、心得、評分必填）
  - 書籍列表展示
- **Should Have (應該要包含)**：
  - 關鍵字搜尋書籍功能
  - 編輯/刪除個別筆記功能
- **Nice to Have (加分的延伸功能)**：
  - 上傳與顯示書籍封面圖片
  - 書籍分類（例如：小說、商業理財、科技等標籤）
  - 閱讀數據統計圖表（例如：依評分或是閱讀書籍數量作長條圖）
