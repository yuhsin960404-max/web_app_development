# PRD - 讀書筆記本系統

## 1. 專案概述
- **背景與動機**：現代人閱讀量大，但看完書後往往容易忘記內容與啟發。讀者需要一個輕量化、專注於記錄的系統，來存放閱讀過的書籍清單、核心心得以及個人評價，協助深化閱讀記憶。
- **目標用戶**：喜愛閱讀的學生、上班族、或是需要整理閱讀清單並記錄心得的愛書人。
- **核心價值主張**：提供一個直覺且簡單的平台，讓使用者能快速記錄書籍、撰寫筆記與評分，並能透過搜尋功能瞬間找回過去的閱讀記憶。

## 2. 功能需求
系統的主要功能與使用者故事如下：

1. **記錄書名與基本資訊**
   - 作為 **讀者**，我希望 **能新增書籍的書名、作者等基本資訊**，以便 **建立並豐富我的個人閱讀清單**。
2. **記錄閱讀心得**
   - 作為 **讀者**，我希望 **能針對每本書撰寫多段落的讀書心得與重點筆記**，以便 **紀錄閱讀時的啟發與重要知識**。
3. **評分系統**
   - 作為 **讀者**，我希望 **能對書籍給予 1 到 5 顆星的主觀評分**，以便 **未來能快速篩選出我最推薦的好書**。
4. **書籍搜尋功能**
   - 作為 **讀者**，我希望 **能用關鍵字搜尋書名或心得內容**，以便 **在忘記具體書名時仍能輕鬆找到相關筆記**。
5. **書籍列表瀏覽與管理**
   - 作為 **讀者**，我希望 **能瀏覽所有紀錄的書籍清單（支援按評分或時間排序）**，以便 **總覽我的閱讀歷史**。
6. **編輯與刪除紀錄**
   - 作為 **讀者**，我希望 **能隨時修改錯誤的紀錄或刪除不需要的書籍**，以便 **維護筆記庫的整潔與正確性**。

## 3. 非功能需求
- **技術限制**：
  - 後端：Flask
  - 前端：Jinja2 模板引擎與原生 HTML/CSS 
  - 資料庫：SQLite
- **效能與安全考量**：
  - 確保資料庫查詢及文字搜尋的效率，維持即使有數千筆紀錄也能快速回應。
  - 對使用者輸入的表單資料進行驗證與過濾（防範基礎的 SQL Injection 及 XSS 攻擊）。
  - 介面需具備基本的 RWD（響應式設計），讓使用者在手機或電腦都能舒適地記錄心得。

## 4. MVP 範圍
- **Must Have (必須有)**：
  - 新增書籍筆記（包含書名、心得、評分）
  - 瀏覽所有書籍列表的頁面
  - 單本書籍詳細內容的檢視頁面
- **Should Have (應該有)**：
  - 關鍵字搜尋（可搜尋書名或心得）
  - 修改與刪除現有的書籍筆記
- **Nice to Have (可以有)**：
  - 標籤分類（例如：小說、商業、科技）
  - 閱讀進度追蹤（未讀、閱讀中、已讀完）
